# Professional Dashboard
The professional dashboard serves as the central hub for professionals to manage appointments, clients, and settings within the Odys frontend. This page details the dashboard's sidebar navigation, its sub-route structure, and the available appointment actions.

## Sidebar
The dashboard sidebar, implemented in `src/app/dashboard/sidebar.tsx`, provides primary navigation to various sections. It displays the professional's name, initials, and current plan information, derived from the `professional` object passed from `src/app/dashboard/layout.tsx`. The navigation links are defined in the `NAV` array within `src/app/dashboard/sidebar.tsx`, including routes for "Dashboard", "Clientes", "Mensagens", "Relatórios", "Assistente IA", "Planos", and "Configurações".

## Sub-routes
The dashboard is structured to support multiple sub-routes, each providing specific functionality. These sub-routes are accessed via the sidebar navigation, with their paths and labels defined in the `NAV` array in `src/app/dashboard/sidebar.tsx`. Examples include the "Clientes" and "Mensagens" sections. The main dashboard content for the root `/dashboard` path is handled by `src/app/dashboard/page.tsx`.

## Appointment Actions
Appointment actions are a core feature, allowing professionals to manage the status of their bookings. These actions, such as confirm, reject, cancel, paid, complete, and no-show, are encapsulated within the `AppointmentActions` component, which is imported and utilized in `src/app/dashboard/page.tsx`. Each action is designed to update the appointment status and trigger relevant system updates. The `STATUS_LABEL` mapping in `src/app/dashboard/page.tsx` defines the display properties for each appointment status.

## Dashboard Structure and Data Flow
The dashboard's overall structure is managed by `src/app/dashboard/layout.tsx`, an async Server Component. This layout performs server-side authentication by calling `createClient()` from `@/lib/supabase/server` and fetching the `user` and `professional` data from `src/lib/db`. If no user or professional is found, it redirects to `/login` or `/onboarding` respectively. The `Sidebar` component in `src/app/dashboard/sidebar.tsx` receives `professionalName`, `professionalSlug`, `plan`, and `trialEndsAt` as props from this layout.

The main dashboard content, rendered by `src/app/dashboard/page.tsx`, is also an async Server Component. It fetches specific data for the dashboard view, including today's and upcoming appointments, total clients, and monthly appointment counts, using `db` from `src/lib/db` and `createClient()` from `@/lib/supabase/server`. This server-side data fetching ensures that the initial dashboard view is populated with up-to-date information before client-side hydration.

## Known gaps
A current trade-off in the dashboard's implementation is the absence of an explicit loading indicator during the initial data fetch for `src/app/dashboard/page.tsx`. As an async Server Component, the page waits for all necessary data (appointments, client counts, plan limits) to be retrieved from the database before rendering. This can result in a brief period where the dashboard appears blank or unresponsive to the user, potentially impacting perceived performance. Implementing a loading state or leveraging Next.js Suspense boundaries could provide better visual feedback during this period.

## Why this shape
The dashboard is designed to provide a clear and intuitive interface for professionals to manage their operations. The use of a dedicated sidebar for navigation, as defined in `src/app/dashboard/sidebar.tsx`, helps organize various features into distinct sub-routes, improving discoverability and user flow. The server-side rendering approach for `src/app/dashboard/layout.tsx` and `src/app/dashboard/page.tsx` ensures that critical data is loaded and authenticated before the page is sent to the client, contributing to a faster initial load time and a more secure environment. Appointment actions are designed for directness and ease of use, reflecting the need for efficient management of daily tasks. This overall structure prioritizes a user-friendly and efficient experience for professionals.
