# PIX BR Code Generation

This page details the implementation of PIX EMV BR Code generation, including CRC16 calculation, and how these codes are rendered as QR codes for client payments.

## PIX EMV BR Code Payload Generation

The core logic for generating the PIX EMV BR Code payload resides in `src/lib/pix.ts`. This module implements the specifications from the Banco Central do Brasil's Manual de Padrões para Iniciação do Pix, specifically for static QR codes.

The `buildPixPayload` function constructs the complete EMV BR Code string. It takes the PIX key, professional's name, city, and an optional amount (in cents) as input.

```typescript
export function buildPixPayload({
  key,
  name,
  city = "Brasil",
  amount,
}: {
  key: string
  name: string
  city?: string
  amount?: number // in cents, e.g. 8000 = R$80
}): string {
  // ... implementation ...
}
```

This function normalizes the `name` and `city` inputs, removing diacritics and truncating them to fit the EMV standard's length constraints. It then assembles various fields according to the PIX specification, such as:
-   `00`: Payload Format Indicator (`01`)
-   `26`: Merchant Account Information (containing `br.gov.bcb.pix` and the PIX `key`)
-   `52`: Merchant Category Code (`0000`)
-   `53`: Transaction Currency (`986` for BRL)
-   `54`: Transaction Amount (if provided, formatted to two decimal places)
-   `58`: Country Code (`BR`)
-   `59`: Merchant Name
-   `60`: Merchant City
-   `62`: Additional Data Field Template (currently `***` for transaction ID)
-   `6304`: CRC placeholder

Each field is formatted using the `field` helper function, which prepends the field ID and its two-digit length to the value.

### CRC16 Calculation

A crucial part of the EMV BR Code standard is the CRC16 checksum, which ensures data integrity. The `crc16` function in `src/lib/pix.ts` calculates this checksum for the entire payload string.

```typescript
function crc16(str: string): string {
  let crc = 0xffff
  for (let i = 0; i < str.length; i++) {
    crc ^= str.charCodeAt(i) << 8
    for (let j = 0; j < 8; j++) {
      crc = crc & 0x8000 ? (crc << 1) ^ 0x1021 : crc << 1
    }
  }
  return (crc & 0xffff).toString(16).toUpperCase().padStart(4, "0")
}
```

The calculated 4-character hexadecimal CRC16 value is appended to the payload string after the `6304` placeholder, completing the EMV BR Code.

## QR Code Rendering

The generated PIX payload is rendered as a scannable QR code using the `PixQR` React component, located at `src/components/pix-qr.tsx`. This component is a client-side component, indicated by `"use client"`.

The `PixQR` component takes `pixKey`, `pixKeyType`, `professionalName`, `amount`, and `paymentType` as props. It uses the `buildPixPayload` function from `src/lib/pix.ts` to generate the EMV BR Code string. The `amount` is included in the payload only if `paymentType` is not `"after"` and a positive `amount` is provided, allowing for both fixed-amount and open-amount PIX payments.

```typescript
const payload = buildPixPayload({
  key: pixKey,
  name: professionalName,
  amount: includeAmount ? amount : undefined,
})
```

The actual QR code image is generated using the `QRCodeSVG` component from the `qrcode.react` library, which takes the `payload` string as its `value` prop.

The component also provides a "Copy PIX code" button. The `handleCopy` function uses the browser's `navigator.clipboard.writeText` API to copy the generated `payload` string to the user's clipboard, offering an alternative to scanning the QR code. The `PIX_TYPE_LABELS` object is used to display a user-friendly label for the PIX key type (e.g., "Telefone" for a phone number key).

## Why this shape

The design separates the pure business logic of PIX EMV BR Code generation from its presentation.
-   `src/lib/pix.ts` encapsulates the technical details of the PIX specification, including field formatting and CRC16 calculation. This makes the payload generation logic reusable and testable independently of any UI framework.
-   `src/components/pix-qr.tsx` is a client-side React component responsible solely for rendering the generated payload as a QR code and providing user interaction (like copying the code). This approach avoids the need for server-side image generation, simplifying deployment and reducing server load.
-   Leveraging standard libraries like `qrcode.react` for QR rendering and browser APIs like `navigator.clipboard.writeText` for copying ensures robust and widely supported functionality without reinventing common patterns.
