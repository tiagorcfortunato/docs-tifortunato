# PIX BR Code Generation

This page covers the implementation of PIX EMV BR Code generation, including payload construction, CRC16 checksum calculation, and QR code rendering using client-side React components.

## PIX EMV BR Code Payload Generation

The `buildPixPayload` function in `src/lib/pix.ts` generates a valid EMV BR Code string for static QR codes, conforming to the Banco Central do Brasil's specification.

It accepts a `key` (PIX key), `name` (merchant name), optional `city` (defaults to "Brasil"), and optional `amount` in cents. The function normalizes `name` and `city` by removing diacritics via Unicode normalization (`NFD`) and stripping combining characters (`[\u0300-\u036f]`). The normalized values are truncated to 25 characters for `name` and 15 for `city`, then converted to uppercase.

Fields are assembled using the `field(id, value)` helper, which formats each segment as `${id}${length}${value}` with a two-digit length prefix. The payload includes:

- `00`: Payload Format Indicator (`01`)
- `26`: Merchant Account Information (subfields `00` = `br.gov.bcb.pix`, `01` = PIX key)
- `52`: Merchant Category Code (`0000`)
- `53`: Transaction Currency (`986` for BRL)
- `54`: Transaction Amount (included only if `amount` is provided and > 0, formatted as `(amount / 100).toFixed(2)`)
- `58`: Country Code (`BR`)
- `59`: Merchant Name
- `60`: Merchant City
- `62`: Additional Data Field Template (`05` = `***` placeholder for transaction ID)
- `6304`: CRC placeholder (replaced after checksum calculation)

The final payload is the concatenation of all parts, with the CRC16 checksum appended.

## CRC16 Checksum Calculation

The `crc16(str)` function in `src/lib/pix.ts` computes the CRC-16-CCITT checksum used in the EMV standard.

It initializes `crc` to `0xffff`. For each character in the input string:
- XOR the character code (shifted left by 8 bits) into `crc`
- Perform 8 bits of feedback shift using polynomial `0x1021`

The result is masked to 16 bits, converted to uppercase hexadecimal, and zero-padded to 4 characters. This value is appended directly to the payload string after the `6304` placeholder.

The CRC is calculated over the entire payload **before** the checksum is added, ensuring compliance with the EMV standard.

## QR Code Rendering

The `PixQR` component in `src/components/pix-qr.tsx` renders the generated PIX payload as a scannable QR code.

It is a client-side React component (`"use client"`) that takes `pixKey`, `pixKeyType`, `professionalName`, `amount`, and `paymentType` as props. The `includeAmount` flag determines whether to pass the `amount` to `buildPixPayload`, based on:
- `paymentType` is not `"after"`
- `amount` is defined and greater than 0

The resulting `payload` string is passed to `QRCodeSVG` from `qrcode.react`, rendered at `size={160}` within a styled container.

A "Copy PIX code" button uses `navigator.clipboard.writeText(payload)` to copy the full EMV BR Code to the clipboard. The `copied` state provides user feedback for 2 seconds.

The component displays:
- A human-readable label for the PIX key type using `PIX_TYPE_LABELS`
- The raw `pixKey` in a monospace, selectable font
- A summary line showing the amount (if applicable) and professional name

## Why this shape

The implementation separates concerns between business logic and presentation:
- `src/lib/pix.ts` contains pure functions for EMV BR Code generation and CRC16 calculation, making it framework-agnostic and easily testable
- The payload format strictly follows the BCB specification without external dependencies
- `PixQR` leverages a well-maintained QR rendering library (`qrcode.react`) and browser-native clipboard API, avoiding server-side image generation and reducing infrastructure complexity
- Conditional amount inclusion supports both fixed and open-amount PIX flows within the same component

This design ensures correctness at the protocol level while maintaining flexibility in UI integration.
