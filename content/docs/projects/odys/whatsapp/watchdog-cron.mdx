## Design Decisions

The decision to implement a dedicated cron job for a WhatsApp watchdog stems from the inherent unreliability of external APIs and the critical nature of client communication. Relying solely on real-time error handling might not catch all issues, especially those related to intermittent connectivity or API service degradation.

1.  **Automated Proactive Monitoring**: By scheduling the watchdog to run daily at `0 9 * * *`, the system adopts a proactive stance. Instead of waiting for user reports or failed message deliveries, the system attempts to identify and potentially rectify issues before they impact users. This daily cadence is a trade-off between real-time monitoring (which can be resource-intensive) and infrequent checks, aiming for a balance that provides timely insights without excessive overhead.
2.  **External API Dependency**: The Evolution API is an external dependency. Its health and responsiveness are outside the direct control of the Odys application. A dedicated watchdog isolates the monitoring of this specific dependency, preventing its failures from silently affecting the entire communication flow.
