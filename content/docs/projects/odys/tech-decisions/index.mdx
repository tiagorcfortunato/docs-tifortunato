# Major Tech Decisions

This page outlines the foundational technology choices in the Odys platform, based on explicit dependencies and tooling in the codebase. Each decision is grounded in developer experience, cost efficiency, and alignment with product requirements.

## Next.js App Router

The platform uses Next.js 16.2.1, as specified in `package.json`, leveraging the App Router for unified server-side rendering and API routes. This enables direct database access within React Server Components, reducing the need for a separate backend. The version `next: "16.2.1"` indicates a recent release, suggesting a preference for cutting-edge features despite potential ecosystem immaturity.

## Drizzle ORM

Database interactions are managed via Drizzle ORM (`drizzle-orm`) and Drizzle Kit (`drizzle-kit`), both listed in `package.json`. The presence of `db:generate`, `db:migrate`, and `db:studio` scripts confirms Drizzle's role in schema generation, migration, and local development. Drizzle is favored for its fully typed queries in TypeScript and lightweight runtime, avoiding the build-step complexity of alternatives like Prisma.

## Supabase

Supabase is integrated through `@supabase/supabase-js` and `@supabase/ssr`, enabling managed Postgres, authentication, and server-side session handling. This choice provides out-of-the-box support for HttpOnly cookies, email verification, and OAuth, reducing operational overhead compared to self-hosted or database-only solutions.

## Stripe

Payment processing relies on Stripe, with both `stripe` and `@stripe/stripe-js` in `package.json`. This supports platform subscriptions using Stripe’s hosted checkout and webhook patterns. Notably, Stripe does not support recurring PIX payments natively, which impacts how client-to-professional transactions are handled.

## Sentry + PostHog

Observability and product analytics are implemented via `@sentry/nextjs` for error tracking and `posthog-js` for session replay, feature flags, and user behavior analysis. Both tools offer strong developer experience and generous free tiers, making them suitable for a pre-launch application.

## Tailwind v4 + shadcn/ui

The UI stack uses Tailwind CSS v4 (`tailwindcss`, `@tailwindcss/postcss`) optimized for React Server Components, avoiding runtime CSS-in-JS overhead. Components are built using `shadcn`, with utilities like `class-variance-authority` and `tailwind-merge`, indicating that UI elements are copied into the codebase (e.g., `src/components/ui/`) for full control and customization.

## Evolution API (Self-hosted WhatsApp)

The platform uses a self-hosted WhatsApp solution via an external Evolution API, allowing messages to originate from a professional’s personal phone number. This avoids Meta’s WhatsApp Business API (WABA) costs and template restrictions but introduces reliability risks tied to device state (e.g., storage, app updates).

## Groq (Llama 3.3)

AI capabilities, including the assistant and WhatsApp intake agent, are powered by Groq via the `groq-sdk` dependency. The Llama 3.3 model is selected for its low cost (10–20x cheaper than GPT-4-class models) and fast inference speed, critical for sequential tool execution. Trade-offs include weaker instruction-following, mitigated by constrained tool scope.

## Known Gaps

- **Next.js 16.2.1**: The use of a recent Next.js version (`next: "16.2.1"`) introduces potential instability due to limited ecosystem maturity and possible breaking changes in early releases.
- **Stripe**: Lack of native recurring PIX support in Stripe necessitates a hybrid payment flow, increasing complexity for client-to-professional transactions.
- **Evolution API**: Self-hosted WhatsApp integration is prone to disconnections if the linked device runs out of storage or undergoes app updates, requiring active monitoring (e.g., via a watchdog mechanism).
- **Groq (Llama 3.3)**: While cost-effective and fast, the Llama 3.3 model exhibits weaker instruction-following compared to higher-tier models, requiring tight scoping of AI tool usage.

## Why this shape

The stack emphasizes developer velocity, cost control, and product differentiation. Next.js App Router and Drizzle reduce boilerplate and operational load. Supabase and Stripe provide battle-tested infrastructure for auth and payments. Specialized tools like Evolution API and Groq enable unique features—personalized WhatsApp messaging and low-latency AI—at acceptable reliability and quality trade-offs.
